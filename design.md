# 亲友圈 - 亲戚关系管理应用设计文档

## 项目概述

一个纯移动端Web应用，用于记录和管理亲戚朋友关系，支持本地存储，无需服务器。

---

## 技术栈

- **框架**: React + TypeScript
- **存储**: IndexedDB dexie(本地浏览器存储)
- **构建**: Vite
- **样式**: Tailwind CSS
- **图表**: D3.js 或 react-flow (关系网络图)
- **图片处理**: react-cropper (照片裁剪)

---

## 核心功能

### 1. 人员管理

**功能描述:**

- 添加、编辑、删除亲友信息
- 信息包括：照片、姓名、关系称呼、电话、生日、备注

**数据模型:**

```typescript
interface Person {
  id: string;
  name: string; // 姓名
  photo?: string; // Base64编码的照片
  callMe?: string; // 对方称呼我（如"侄子"）
  iCall?: string; // 我称呼对方（如"叔叔"）
  phone?: string; // 电话
  birthday?: string; // 生日 (YYYY-MM-DD)
  notes?: string; // 备注
  createdAt: number; // 创建时间戳
  updatedAt: number; // 更新时间戳
}

interface Relationship {
  id: string;
  fromPersonId: string; // 关系起点人员ID
  toPersonId: string; // 关系终点人员ID
  relationLabel: string; // 关系标签（如"父子"、"夫妻"）
}
```

### 2. 视图切换

**页面结构:**

- 单一页面，顶部Tab切换"列表视图"和"关系图视图"
- 无底部导航栏

**列表视图:**

- 人员卡片列表（头像+姓名+称呼）
- 支持搜索和筛选
- 点击卡片进入详情页

**关系图视图:**

- 可缩放、平移的网络拓扑图
- 节点：圆形头像+姓名
- 连线：关系称呼标签
- 点击节点弹出详情浮层

### 3. 搜索筛选

**搜索功能:**

- 顶部搜索框，实时搜索姓名

**筛选功能:**

- 按关系类型筛选（如只看父亲那边的亲戚）
- 按姓氏筛选

### 4. 照片处理

**功能:**

- 从相册选择照片
- 圆形裁剪编辑
- Base64存储在IndexedDB

### 5. 关系建立

**添加人员时:**

- 可以同时指定与已有人员的关系
- 手动输入关系称呼（如"表哥"、"婶婶"）

### 6. 数据导出/导入

**导出:**

- 将所有人员和关系导出为JSON
- 照片使用Base64嵌入JSON中（单文件）

**导入:**

- 选择JSON文件恢复数据
- 完全覆盖现有数据

## 技术实现要点

### 1. 本地存储 (IndexedDB)

使用 Dexie.js 简化 IndexedDB 操作：

```typescript
import Dexie from "dexie";

class RelationshipDB extends Dexie {
  persons!: Dexie.Table<Person, string>;
  relationships!: Dexie.Table<Relationship, string>;

  constructor() {
    super("RelationshipDB");
    this.version(1).stores({
      persons: "id, name, createdAt",
      relationships: "id, fromPersonId, toPersonId",
    });
  }
}
```

### 2. 关系图渲染

使用 D3.js 实现力导向图：

- 节点：圆形头像 + 姓名标签
- 连线：带关系标签的线条
- 交互：
  - 双指缩放
  - 单指平移
  - 点击节点显示详情

### 3. 照片处理

- 使用 `<input type="file" accept="image/*">` 选择照片
- 使用 react-cropper 进行圆形裁剪
- 使用 canvas 压缩图片尺寸
- Base64 存储（建议限制单张不超过 500KB）

### 4. 导出/导入

**导出:**

```typescript
const exportData = {
  version: "1.0",
  exportDate: Date.now(),
  persons: await db.persons.toArray(),
  relationships: await db.relationships.toArray(),
};
const blob = new Blob([JSON.stringify(exportData)], {
  type: "application/json",
});
// 触发下载
```

**导入:**

```typescript
const data = JSON.parse(await file.text());
await db.persons.clear();
await db.relationships.clear();
await db.persons.bulkAdd(data.persons);
await db.relationships.bulkAdd(data.relationships);
```

## 性能考虑

1. **照片压缩**: 上传时压缩至最大 500x500px，质量 0.8
2. **虚拟列表**: 人员列表使用虚拟滚动（超过50人时）
3. **防抖搜索**: 搜索输入防抖 300ms
4. **关系图优化**: 节点超过100个时启用性能模式（简化渲染）
